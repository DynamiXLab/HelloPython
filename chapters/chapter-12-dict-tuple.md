### 12.1 字典——用"名字"来找数据

列表用数字索引，字典用**键（key）**来索引**值（value）**。字典的每个元素都是一个**键值对**（key-value pair）。

```python
student = {
    "name": "小明",      # 键"name" → 值"小明"
    "age": 12,           # 键"age"  → 值12
    "city": "北京"       # 键"city" → 值"北京"
}

print(student["name"])    # 小明
print(student["age"])     # 12
```

可以把字典理解成一个**有名字标签的收纳盒**：列表是"第 0 格放什么、第 1 格放什么"，字典是"名字叫'name'的格子里放'小明'"。

#### 字典的结构

字典把数据分成两大列：**键**和**值**。键是查找条件，值是查到的结果：

```
键（key）       值（value）
───────       ──────────
"name"    →    "小明"
"age"     →    12
"city"    →    "北京"
```

- 键必须是**唯一的**——不能有两个相同的键
- 键必须是**不可变类型**——字符串、数字、元组可以当键，**列表不能当键**
- 值可以是任意类型——数字、字符串、列表、甚至另一个字典

#### 字典的常用操作

```python
student = {"name": "小明", "age": 12, "city": "北京"}

# 新增或修改
student["score"] = 95     # 新增一个键值对
student["age"] = 13       # 修改已有的值（键"age"已存在）

# 判断键是否存在
print("name" in student)   # True
print("grade" in student)  # False

# 安全取值——不存在时返回默认值，不报错
print(student.get("name"))         # 小明
print(student.get("grade"))        # None（不存在，不报错）
print(student.get("grade", "无"))  # "无"（指定默认值）

# 获取所有键/所有值/所有键值对
print(student.keys())     # dict_keys(['name', 'age', 'city', 'score'])
print(student.values())   # dict_values(['小明', 13, '北京', 95])
print(student.items())    # dict_items([('name', '小明'), ('age', 13), ...])

# 删除
del student["city"]       # 删除键"city"及其对应的值
```

#### 查键用 `in`，别用手动遍历

查字典里有没有某个键，最地道的写法是 `键 in 字典`，而不是自己写循环：

```python
# ✅ 正确
if "name" in student:
    print(student["name"])

# ❌ 没必要——字典不是列表，不能遍历着查
```

#### 字典嵌套

字典的值可以是另一个字典，这是处理复杂数据的常用结构：

```python
classroom = {
    "小明": {"chinese": 92, "math": 88},
    "小红": {"chinese": 85, "math": 95}
}

print(classroom["小明"]["math"])   # 88
```

### 12.2 遍历字典

```python
student = {"name": "小明", "age": 12, "city": "北京"}

# 遍历键
for key in student:
    print(key)           # name → age → city

# 遍历键值对
for key, value in student.items():
    print(f"{key}: {value}")
# name: 小明
# age: 12
# city: 北京

# 只遍历值
for value in student.values():
    print(value)         # 小明 → 12 → 北京
```

### 12.3 元组——不可修改的列表

元组用 `()` 定义，一旦创建就不能修改（增删改都不行）：

```python
colors = ("红", "黄", "蓝")
print(colors[0])       # 红
print(colors[1:3])     # ("黄", "蓝")  ← 元组也支持切片
# colors[0] = "绿"     # ❌ TypeError：元组不可修改
```

**为什么需要元组？**

有些数据天生不应该被修改——比如一周七天、RGB 颜色值、身份证号。用元组存这些数据，等于给代码加了一道保险：万一有人想改，Python 会直接报错，而不是悄无声息地改掉一个不该动的值。

**元组 vs 列表的异同**

| 对比 | 列表 | 元组 |
|------|------|------|
| 定义 | `[]` | `()` |
| 修改 | ✅ 可增删改 | ❌ 不可修改 |
| 切片 | ✅ 支持 | ✅ 支持 |
| 遍历 | ✅ 支持 | ✅ 支持 |
| 内存占 | 较大 | 较小 |
| 作用 | 需要改变的数据 | 不应该改变的数据 |

### 12.4 列表、元组、字典对比

这三者都是 Python 中存储多个数据的容器，各有各的用途：

| 对比项 | 列表（list） | 元组（tuple） | 字典（dict） |
|-------|------------|-------------|------------|
| 定义符号 | `[]` | `()` | `{}` |
| 访问方式 | 数字索引 `[0]` | 数字索引 `[0]` | 键索引 `["name"]` |
| 是否可修改 | ✅ 可增删改 | ❌ 不可修改 | ✅ 可增删改 |
| 元素是否有序 | ✅ 有序 | ✅ 有序 | ❌ 无序（Python 3.7+ 记插入顺序） |
| 元素是否唯一 | 可重复 | 可重复 | **键唯一**，值可重复 |
| 查找速度 | 慢（从头挨个找） | 慢（从头挨个找） | **极快**（哈希直接定位） |
| 内存占用 | 少 | 最少 | 多（预分配空间） |
| 适用场景 | 有序数据集合 | 不该改的数据 | 需要快速查找的键值映射 |

**怎么选？**

- 需要按位置顺序存数据 → **列表**
- 数据确定了不会再变 → **元组**（更安全、更省内存）
- 需要用名字（而不是数字）来查数据 → **字典**
- 需要快速知道某个值是否存在 → **字典**

💡 **思考延续**

你有没有好奇过：字典为什么查询那么快？写 `student["name"]` 的时候，Python 到底做了什么？

如果让你在列表 `[("小红", 88), ("小明", 95), ("小刚", 78)]` 中找"小刚"的成绩，你怎么做？只能从头挨个比较——"是小红吗？不是。是小明吗？不是。是小刚吗？是的，找到了。"这种查找方式叫线性查找，如果列表里有 10000 个人，最坏情况要比较 10000 次才能找到。

但字典不一样。它的背后是一种叫**哈希表**的数据结构。当你在字典中查找 `student["小刚"]` 时，Python 不会挨个去比——它用了一个数学函数（哈希函数）对键名 "小刚" 做一次快速计算，直接算出这个键对应的值存放在内存的哪个位置，然后一步到位取出来。不管字典里有 10000 还是 100000 条数据，找到一条的速度基本不变。计算机科学里管这个叫 O(1) 时间复杂度。

代价是什么？内存。列表几乎不浪费额外空间，而字典需要预先分配一大块空白区域来存放"哈希计算结果"。这就是计算机科学中最经典的一个取舍——**用空间换时间**。你写程序的时候，只要知道什么时候该用列表（顺序遍历、节省内存），什么时候该用字典（快速查找），就已经掌握了这个取舍的精髓。
