### 5.1 字符串的定义

字符串就是一串字符，用单引号或双引号包裹：

```python
s1 = 'hello'
s2 = "你好"
s3 = "It's a book"    # 内容里有单引号，外面用双引号
```

### 5.2 字符串拼接

```python
first = "张"
last = "三"
name = last + first    # "三张"
greeting = "你好，" + name + "！"
print(greeting)        # 你好，三张！
```

### 5.3 f-string——更优雅的拼接方式

前面我们用 `+` 来拼接字符串：

```python
name = "小明"
age = 12
print("我叫" + name + "，今年" + str(age) + "岁")
```

这样写容易漏掉引号和 `+`，还要手动转类型（`str(age)`）。**f-string** 可以帮你省掉这些麻烦：

```python
name = "小明"
age = 12
print(f"我叫{name}，今年{age}岁")  # 我叫小明，今年12岁
```

`f` 是 format（格式化）的缩写，**写在引号前面，紧贴着引号，中间不留空格**。`{变量名}` 就像占位符，Python 会在运行时把变量的值填进去。

> ⚠️ **新手最容易犯的错**：写了花括号 `{}`，但忘了在引号前面加 `f`。这时 Python 不会报错，但它会把 `{name}` 当成普通文本原样输出——你看到的就是 `"我叫{name}"` 而不是 `"我叫小明"`。养成习惯：**看到花括号就先确认前面有没有 `f`**。

**f-string 的本质：花括号里是一小段 Python 代码**

f-string 的花括号 `{}` 不是简单的\"把变量名替换成值\"——它实际上会计算里面写的一小段 Python 表达式，然后把计算结果转成字符串填进去。

变量名是最简单的形式，但你可以写计算、调函数、甚至做更复杂的事情：

```python
# 直接写变量
name = "小明"
print(f"你好，{name}")          # 你好，小明

# 写计算表达式
print(f"2 的 10 次方 = {2 ** 10}")  # 2 的 10 次方 = 1024

# 调用函数
s = "  hello  "
print(f"去掉空格后：{s.strip()}")   # 去掉空格后：hello

# 多个表达式混合
a, b = 15, 4
print(f"{a} ÷ {b} = {a // b} 余 {a % b}")  # 15 ÷ 4 = 3 余 3
```

但要注意：花括号里不能写多行语句（比如 `if`、`for`），也不能赋值。f-string 的设计原则是\"只计算，不做事\"——适合快速格式化，不适合放复杂逻辑。

**数字格式化：`:.2f` 的来龙去脉**

`:2f` 这部分是**格式说明符**，跟在表达式后面用冒号分隔：

| 写法 | 含义 | 示例 | 输出 |
|------|------|------|------|
| `:.2f` | 保留 2 位小数 | `f"{3.14159:.2f}"` | `3.14` |
| `:.0f` | 保留 0 位小数（取整） | `f"{3.14159:.0f}"` | `3` |
| `:>5d` | 整数，右对齐，占 5 格 | `f"{42:>5d}"` | `   42` |
| `:<5d` | 整数，左对齐，占 5 格 | `f"{42:<5d}"` | `42   ` |

```python
pi = 3.1415926
print(f"π ≈ {pi:.2f}")       # π ≈ 3.14
print(f"π ≈ {pi:.4f}")       # π ≈ 3.1416（自动四舍五入）

score = 95
print(f"成绩：{score:>4d}")   # 成绩：  95（右对齐凑数）
```

> 格式说明符的完整规则后面还会遇到，现在先记住 `:.Nf`（保留 N 位小数）就够了。

```python
a, b = 10, 3
print(f"{a} + {b} = {a + b}")          # 10 + 3 = 13
print(f"{a} ÷ {b} = {a / b}")          # 10 ÷ 3 = 3.3333333333...
print(f"{a} ÷ {b} = {a / b:.2f}")      # 10 ÷ 3 = 3.33（保留两位小数）
```

花括号 `{}` 里放的是 Python 表达式——变量、运算、函数调用都可以。`:2f` 是**格式说明符**，`:.2f` 表示保留两位小数，后面学数字格式化时会经常用到。

### 5.4 转义字符——字符串里的特殊符号

有时候你想在字符串里放一些**不能直接输入**的字符——比如换行、制表符（Tab）、或者引号本身。这时候需要**转义字符**：用反斜杠 `\` 加一个字母来表示。

| 转义字符 | 含义 | 示例 | 输出效果 |
|---------|------|------|---------|
| `\n` | 换行 | `"第一行\n第二行"` | 两行显示 |
| `\t` | 制表符（Tab） | `"A\tB"` | `A    B` |
| `\\` | 反斜杠本身 | `"路径：C:\\\\"` | `路径：C:\` |
| `\'` | 单引号 | `'It\'s OK'` | `It's OK` |
| `\"` | 双引号 | `"他说：\"好\""` | `他说："好"` |

试试换行的效果：

```python
print("第一行\n第二行\n第三行")
# 输出：
# 第一行
# 第二行
# 第三行
```

试试制表符对齐：

`\t` 并不是\"固定加 N 个空格\"，它的含义是**跳到下一个制表位**。终端里每个制表位默认相隔 8 个字符，所以 `\t` 会把光标推到下一个 8 的倍数位置：

```python
print("A\tB")
print("AAAAA\tB")
# 输出：
# A       B
# AAAAA   B
```

上面的输出中，`A` 后面到 8 格处的空白是 `\t` 推过去的，`AAAAA` 占了 5 格，`\t` 只推到第 8 格（下一个 8 的倍数），所以这次只补了 3 个空位。

对齐表格时用它最方便——每列内容用 `\t` 隔开，各列的起点会自动对齐到同一位置：

```python
print("姓名\t年龄\t城市")
print("小明\t12\t北京")
print("小红\t13\t上海")
# 输出：
# 姓名    年龄    城市
# 小明    12      北京
# 小红    13      上海
```

如果某列内容超过 8 个字符，`\t` 会跳到 16 格（再下一个 8 的倍数），所以不要太依赖它做严格对齐，单纯做\"各列分开、基本对齐\"绰绰有余。

> **小贴士**：`print()` 默认在末尾加换行，所以不加 `\n` 也会自动换行。`\n` 主要用在"一行里要换行"的时候，比如写入文件。

### 5.5 原始字符串 r''——拒绝转义

有些场景下（比如文件路径、正则表达式），字符串里有很多反斜杠 `\`，如果每个都要写成 `\\` 会很痛苦。

这时候用 **原始字符串（raw string）**——在引号前加 `r`，告诉 Python："里面的反斜杠我就是真想写反斜杠，你别把它当成转义！"

```python
# 普通字符串：每个反斜杠都要双写
path = "C:\\Users\\小明\\Desktop"
print(path)    # C:\Users\小明\Desktop

# 原始字符串：写一个反斜杠就够了
path = r"C:\Users\小明\Desktop"
print(path)    # C:\Users\小明\Desktop
```

另一个例子——打印带 `\n` 的文本，不想要它真的换行：

```python
text1 = "第一行\n第二行"      # 普通字符串：\n 被当作换行
print(text1)
# 第一行
# 第二行

text2 = r"第一行\n第二行"     # 原始字符串：\n 就是两个字符
print(text2)
# 第一行\n第二行
```

> **总结对比**：
> - `f"..."` — 花括号 `{}` 里写表达式，自动替换
> - `r"..."` — 反斜杠就是反斜杠，不做转义
> - 两个可以混用：`rf"{变量}\n{变量}"`（f 和 r 的顺序任意）

### 5.6 常用字符串方法速查

Python 的字符串自带很多方法，可以直接通过 `字符串.方法名()` 来调用。以下列出最常用的几个——练习任务里会用到它们：

| 方法 | 作用 | 示例 | 返回结果 |
|------|------|------|---------|
| `strip()` | 去掉首尾空格 | `"  hi  ".strip()` | `"hi"` |
| `upper()` | 全部转大写 | `"Hi".upper()` | `"HI"` |
| `lower()` | 全部转小写 | `"Hi".lower()` | `"hi"` |
| `title()` | 每个单词首字母大写 | `"hello world".title()` | `"Hello World"` |
| `replace(a, b)` | 把 a 替换成 b | `"hi hi".replace("hi", "ok")`| `"ok ok"` |
| `count(x)` | 统计 x 出现次数 | `"a b a".count("a")` | `2` |

不是方法但常用的字符串相关函数：

| 函数 | 作用 | 示例 | 返回结果 |
|------|------|------|---------|
| `len(s)` | 获取字符串长度 | `len("你好")` | `2` |

试试几个：

```python
text = "  Hello Python  "
print(text.strip())                              # Hello Python
print(text.upper())                              #   HELLO PYTHON  
print("hello world".title())                     # Hello World
print("hello hello".replace("hello", "hi"))      # hi hi
print("apple, banana, apple".count("apple"))     # 2
print(len("Python"))                             # 6
```

> 方法名后面的括号里可以传参，比如 `replace("旧", "新")` 里的 `"旧"` 和 `"新"` 就是参数。后面学函数时会正式介绍参数的概念。

💡 **思考延续**

你刚学的字符串看起来就是"一堆文字"，但其实在计算机诞生之初，根本不存在"文字"——机器只认数字。要让计算机显示文字，就必须给每个字符编一个编号，这就是"字符编码"的由来。

最早通行的是 **ASCII** 码表（1963 年发布），只有 128 个字符：大小写英文字母、数字 0~9、标点符号和一些控制字符，每个占 1 个字节。比如大写 `A` 是 65，小写 `a` 是 97——你看到的是字母，计算机看到的只是数字 65 和 97。

但问题来了：ASCII 只有 128 个位置，装下英文还行，中文怎么办？汉字有几万个，1 个字节根本不够。于是各个国家各自为政：中国有 GB2312、日本有 Shift-JIS……这就导致了一个困境：你用 GB2312 保存的文件，换一台用 Shift-JIS 的电脑打开就是乱码。

直到 1990 年代，一个叫 **Unicode** 的全球统一编码标准出现，它用一个超大号的表格给地球上所有已知书写系统的每个字符分配了一个唯一的编号——中文的"中"是 20013，阿拉伯字母、藏文、甚至 Emoji 都有。现在 Python 3 默认使用 Unicode 存储字符串，你在代码里写 `"你好"` ，Python 知道它的编号分别是 20320 和 22909。`print()` 帮你把数字翻译回文字显示在屏幕上，所以你从来不需要操心这层转换。

下次看到 Emoji 的时候可以想一想：那个 😂 在计算机里也只是一个编号（128514），是你的屏幕把它翻译回了图案。
