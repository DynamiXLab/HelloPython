### 8.1 什么是列表

列表用 `[]` 表示，里面可以放多个数据，用逗号隔开：

```python
fruits = ["苹果", "香蕉", "橘子", "葡萄"]
scores = [85, 92, 78, 60, 95]
mixed = ["小明", 12, 1.75]          # 可以混合不同类型
```

### 8.2 访问列表元素——用索引

列表里的每个元素都有一个**编号**，这个编号叫**索引**（index）。通过索引可以快速找到对应位置的元素。

```python
fruits = ["苹果", "香蕉", "橘子", "葡萄"]
print(fruits[0])    # 苹果
print(fruits[1])    # 香蕉
print(fruits[2])    # 橘子
print(fruits[3])    # 葡萄
```

#### 索引从 0 开始

你可能很不习惯——**第一个元素的索引是 0，不是 1**。这不是拍脑袋定的，有实际原因。

列表本质上是一段连续的内存空间。如果把列表想象成一排并排的盒子，每个盒子大小一样：

```
位置偏移： 0格    1格    2格    3格
         ┌────┐ ┌────┐ ┌────┐ ┌────┐
         │苹果│ │香蕉│ │橘子│ │葡萄│
         └────┘ └────┘ └────┘ └────┘
索引：     0      1      2      3
```

计算机找元素时，并不是数"第几个"，而是计算**偏移量**（offset）：从起始位置往前走多少步。第一个元素就在起点，偏移 0 步，所以索引是 0；第二个元素偏移 1 步，索引是 1…… 这就是索引从 0 开始的根源——**索引 = 偏移量**，从第一个位置出发的步数。

> 有些语言（如 MATLAB、Lua）索引从 1 开始，但那对计算机来说反而是"需要额外减 1"的操作。Python 沿用了 C 语言的 0 索引传统，这在处理切片和循环时还会体现出更多便利。

#### 每个索引对应一个元素

```python
# 列表的长度是 4，有效索引范围是 0~3
fruits = ["苹果", "香蕉", "橘子", "葡萄"]
print(len(fruits))        # 4
print(fruits[0])          # ✅ 苹果
print(fruits[3])          # ✅ 葡萄
# print(fruits[4])        # ❌ IndexError：索引 4 超出范围（最大是 3）
```

有效索引的范围是 `0` 到 `len(列表) - 1`。超出就报 `IndexError`。

#### 负索引：从末尾往前数

Python 还支持**负索引**，从列表末尾往前数：

```python
fruits = ["苹果", "香蕉", "橘子", "葡萄"]

print(fruits[-1])   # 葡萄  ← 倒数第 1 个
print(fruits[-2])   # 橘子  ← 倒数第 2 个
print(fruits[-3])   # 香蕉
print(fruits[-4])   # 苹果
```

负索引的范围是 `-1` 到 `-len(列表)`。

```
索引：    -4     -3     -2     -1
         ┌────┐ ┌────┐ ┌────┐ ┌────┐
         │苹果│ │香蕉│ │橘子│ │葡萄│
         └────┘ └────┘ └────┘ └────┘
索引：     0      1      2      3
```

可以看到正索引和负索引的关系：`负索引 = 正索引 - len(列表)`，比如 `-1 = 3 - 4`。

> 负索引在"取列表最后几个元素"时非常好用，后面学切片会大量用到。

### 8.3 列表常用操作

```python
nums = [10, 20, 30]

nums.append(40)        # 末尾追加：[10, 20, 30, 40]
nums.remove(20)        # 删除元素 20：[10, 30, 40]
nums.insert(1, 15)     # 在索引1处插入整数15：[10, 15, 30, 40]

print(len(nums))       # 获取长度：4
```

### 8.4 列表切片——取一段子列表

切片是从列表中**取出一段连续的子列表**，语法是 `列表[起点:终点]`。和 `range()` 一样，**起点包含，终点不包含**。

```python
nums = [10, 20, 30, 40, 50]

print(nums[1:4])       # [20, 30, 40]    ← 起点=1，终点=4（不含4）
```

**省略起点或终点**

```python
nums = [10, 20, 30, 40, 50]

print(nums[:3])        # [10, 20, 30]    ← 从开头取到索引3（不含3）
print(nums[2:])        # [30, 40, 50]    ← 从索引2取到末尾
print(nums[:])         # [10, 20, 30, 40, 50]  ← 全部复制一份
```

`nums[:]` 是复制列表的最简洁方式，后面介绍变量引用时会讲到它和直接 `=` 赋值的区别。

**步长——每隔几个取一个**

切片的完整格式是 `列表[起点:终点:步长]`。默认步长是 1（连续取），你可以改：

```python
nums = [10, 20, 30, 40, 50, 60]

print(nums[0:6:2])     # [10, 30, 50]    ← 从0到6，每2个取1个
print(nums[::3])       # [10, 40]        ← 整个列表，每3个取1个
print(nums[::2])       # [10, 30, 50]    ← 取所有偶数索引（第 0,2,4 个）
```

**负索引切片**

还记得负索引从末尾开始数吗？切片也支持：

```python
nums = [10, 20, 30, 40, 50]

print(nums[-3:-1])     # [30, 40]    ← 倒数第3个到倒数第1个（不含）
print(nums[-2:])       # [40, 50]    ← 取最后2个
print(nums[:-2])       # [10, 20, 30]  ← 去掉最后2个
```

**步长为负数——反转列表**

步长为负数时，切片会**反向**遍历：

```python
nums = [10, 20, 30, 40, 50]

print(nums[::-1])      # [50, 40, 30, 20, 10]  ← 整个列表反转
print(nums[4:0:-1])    # [50, 40, 30, 20]      ← 从索引4倒着取到1
print(nums[4::-2])     # [50, 30, 10]           ← 从末尾开始，每2个取1个
```

> 正步长：起点 < 终点（从左往右）；负步长：起点 > 终点（从右往左）。

**切片不报错——比索引安全**

访问单个元素时，索引越界会报错：

```python
nums = [10, 20, 30]
# print(nums[10])      # ❌ IndexError: list index out of range
```

但切片不会——如果超出范围，Python 会默默地**取到尽头为止**：

```python
nums = [10, 20, 30]

print(nums[1:10])      # [20, 30]    ← 终点10超出，自动截断到末尾
print(nums[10:20])     # []          ← 起点超出，返回空列表
print(nums[-10:2])     # [10, 20]    ← 负索引超出，从有效的起点开始
```

**切片常用场景速查**

| 写法 | 含义 | 结果示例 |
|------|------|---------|
| `nums[:3]` | 取前 3 个 | `[10, 20, 30]` |
| `nums[3:]` | 从第 4 个取到末尾 | `[40, 50]` |
| `nums[-2:]` | 取最后 2 个 | `[40, 50]` |
| `nums[:-2]` | 去掉最后 2 个 | `[10, 20, 30]` |
| `nums[:]` | 复制整个列表 | `[10, 20, 30, 40, 50]` |
| `nums[::-1]` | 反转整个列表 | `[50, 40, 30, 20, 10]` |
| `nums[::2]` | 隔一个取一个 | `[10, 30, 50]` |

💡 **思考延续**

你可能已经注意到了：列表的第一个元素是 `fruits[0]`，不是 `fruits[1]`。为什么编程语言不直接"从 1 开始"，像正常人一样数数？

这个问题的标准答案，来自一位脾气出了名大的荷兰计算机科学家——**Edsger W. Dijkstra**。他在 1982 年写了一篇只有两页的短文《为什么编号要从零开始》（"Why numbering should start at zero"），用一段非常 Dijkstra 风格的严密论证给出了结论。

他的核心论据是这样的：假设你要表示"自然数 1 到 10"，有四种方式选择区间表示法：
- `0 < i < 11` — 不好，因为 0 和 11 都不是区间内的数，看起来不直观
- `1 ≤ i ≤ 10` — 看起来自然，但算元素个数时要用 `(10 - 1 + 1) = 10`，多了一步
- `0 ≤ i < 10` — 元素个数就是 `10 - 0 = 10`，一步到位
- `1 < i ≤ 10` — 最差，没有理由用这个

Dijkstra 论证说 `0 ≤ i < 10` 最优雅，因为 (1) 下界不需要加 1；(2) 上界减下界正好是元素个数；(3) 对于空区间，`0 ≤ i < 0` 比 `1 ≤ i ≤ 0` 自然得多。
