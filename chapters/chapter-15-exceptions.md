### 15.1 你一定见过这些红色文字

在本教程前面的练习里，你一定遇到过这种情况：满心期待地点击运行，结果 Thonny 的 Shell 里冒出一段**红色文字**，程序直接停止了。

比如你在第6章写这个：

```python
age = int(input("请输入年龄："))
```

然后手一抖输入了 `十二`，而不是 `12`——屏幕上就会跳出：

```
ValueError: invalid literal for int() with base 10: '十二'
```

再比如第8章：

```python
nums = [10, 20, 30]
print(nums[10])
```

又跳出一段红色：

```
IndexError: list index out of range
```

**这些红色文字叫"错误信息"（或者更准确地说，"异常信息"）。** 编程中遇到报错是**每天都在发生的事情**——老程序员看到的报错次数比新手只多不少。区别在于：老程序员知道怎么读这个信息、怎么处理它。

### 15.2 最常见的几种错误类型

Python 把不同种类的错误分成不同的类型。认识最常见的几种，你就不会每次看到红色文字都慌了：

**`NameError`（名字错误）——变量没定义过**

```python
print(abc)        # ❌ abc 这个名字从来没出现过
```

程序执行到 `print(abc)` 时，Python 翻遍了内存也找不到叫 `abc` 的变量，只好告诉你：这名字我不认识。

**`TypeError`（类型错误）——不同种类的数据不能混用**

```python
"hello" + 5      # ❌ 字符串不能和整数相加
len(123)         # ❌ 整数不能求长度（len 只有字符串、列表、字典能用）
```

这个在第3章讲变量时就见过了——Python 对数据类型有严格的规则，违规了它不会自作主张帮你转，而是直接报错。

**`ValueError`（值错误）——内容本身不合法**

```python
int("abc")       # ❌ "abc" 确实是个值，但它不能转成整数
```

形状（类型）是对的——`int()` 的确收字符串参数——但内容不对："abc" 不是数字格式。

**`IndexError`（索引错误）——索引超出范围**

```python
nums = [1, 2, 3]
print(nums[5])   # ❌ 列表只有 0,1,2，没有索引 5
```

这个在第8章讲列表时也遇到过了——索引超出 `len(列表)-1` 的范围。

> 看到这些错误不用怕。Python 的优点之一就是**报错信息写得很清楚**：它告诉你是哪种错误、发生在哪一行、是什么原因。学会读错误信息，就等于学会了自己调试的一半技能。

### 15.3 try...except——给程序穿件"防弹衣"

前面的所有报错都会让程序**直接崩溃**——屏幕一红，程序停了，后面还有代码也不跑了。

但很多时候，你希望程序**不要崩溃**，而是提示用户重新输入，继续运行。`try...except` 就是干这个的：

```python
try:
    age = int(input("请输入年龄："))
    print(f"你输入的是：{age}")
except ValueError:
    print("输入的不是数字！")
```

**它的工作方式：**
1. Python 先尝试执行 `try` 下面的代码块
2. 如果一切正常，`except` 部分被跳过，继续往后运行
3. 如果 `try` 里面出了 `ValueError` 类型的错误，程序**不会崩溃**，而是立刻跳到 `except ValueError:` 下面的代码执行

```
用户输入 25 → 输出 "25"，顺利结束
用户输入 "十二" → 输出 "输入的不是数字！"，程序没有崩溃
```

可以对比一下不用 `try...except` 的版本——输入 "十二" 时屏幕上直接跳红色错误信息，程序停止。

**捕获多种错误**

一个 `try` 后面可以跟多个 `except`，每种错误用不同的方式处理：

```python
try:
    a = int(input("请输入第一个整数："))
    b = int(input("请输入第二个整数："))
    result = a / b
    print(f"结果是：{result}")
except ValueError:
    print("输入的不是数字！")
except ZeroDivisionError:
    print("除数不能为 0！")
```

输入 `10` 和 `0` → 触发 `ZeroDivisionError` → 显示"除数不能为 0！"

### 15.4 循环 + try...except——直到输入对为止

一个非常常见的模式：用 `while True` 循环让用户反复输入，直到输入合法为止：

```python
while True:
    try:
        num = int(input("请输入一个整数："))
        break    # 输入成功，跳出循环
    except ValueError:
        print("格式错误，请重新输入！")

print(f"你输入的数字是：{num}")
```

这个程序会一直问，直到用户老老实实输入了一个整数才继续往下走。实际项目里这个模式用得非常多——你不可能假设用户第一次就输入正确。

**任务（⭐⭐⭐ 困难）**：写一个"安全文件读取器"：

1. 提示用户输入文件名
2. 尝试读取并显示文件内容
3. 处理所有可能的异常：
   - 文件不存在（`FileNotFoundError`）→ 提示"文件不存在"
   - 权限不足（`PermissionError`）→ 提示"没有读取权限"
   - 编码错误（`UnicodeDecodeError`）→ 提示"文件编码不支持"
   - 其他异常 → 提示"未知错误"并显示错误信息
4. 用 `while True` 循环让用户反复尝试，输入 `quit` 退出

```python
while True:
    filename = input("请输入文件名（quit退出）：")
    if filename == "quit":
        break
    try:
        with open(filename, "r", encoding="utf-8") as f:
            print(f.read())
    except FileNotFoundError:
        print("文件不存在！")
    except PermissionError:
        print("没有读取权限！")
    except UnicodeDecodeError:
        print("文件编码不支持！")
    except Exception as e:
        print(f"未知错误：{e}")
```

💡 **思考延续**

这一章你学会了如何捕获和处理程序中的错误。这背后其实是一种重要的编程哲学：**防御式编程**（Defensive Programming）。

核心思想很简单粗暴：永远假设事情会出问题。用户不会按你期望的方式输入——ta 会在你要求数字的时候输入字母。文件可能不存在——用户提前删了它或者忘记创建了。网络连接可能断开——服务器宕机了、网线松了。

如果你在写程序时默认"一切顺利、用户很乖"，那你的程序就像一个在沙滩上建的城堡——潮水（意外情况）一来就塌了。防御式编程是让你提前给城堡打好地基：不假设用户会输整数，用 `try...except ValueError`；不假设文件一定存在，用 `try...except FileNotFoundError`。

但有一个重要的经验法则：**只捕获你知道该怎么处理的异常**。如果程序遇到了一个你完全没想到的、不知道怎么恢复的错误，不要用一把空的 `try...except` 把它静默掉——这就像听到火灾报警器响了，你去按了静音键继续睡觉。真正应该做的是让它"冒泡"到更高层，要么让更了解情况的代码处理它，要么让程序崩溃并打印出错误信息——至少你知道哪里出了问题。

Python 社区有句话："Errors should never pass silently."（错误永远不该悄无声息地溜走。）学会这个，你的代码就会比 90% 的新手写的要健壮得多。
