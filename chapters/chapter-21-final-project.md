### 21.1 通讯协议——设备之间怎么"说话"

毕业项目要用到的 OLED 屏幕和 AHT20 温湿度传感器都挂在同一条 I2C 总线上。嵌入式设备之间要交换数据，必须遵守一定的"规则"（协议）。这里介绍最常用的四种。

#### I2C——两根线挂一堆设备

I2C（Inter-Integrated Circuit，读作"I方C"，Philips 1982年发明）只用**两根线**就能连接多达 127 个设备：

> 🖼️ 搜「I2C 总线拓扑 SDA SCL」或让 AI 画一张：ESP32 作为主设备，两条线上挂着 OLED（0x3C）和 AHT20 传感器（0x38），SDA/SCL 各有一个上拉电阻接到 3.3V。

- **SDA（Serial Data）**：数据线，双向传输
- **SCL（Serial Clock）**：时钟线，由主机发出节拍来同步通信

每个 I2C 设备有一个**7 位地址**，ESP32 通过地址找到对应的设备。本项目里 OLED 地址是 `0x3C`，AHT20 传感器地址是 `0x38`——两个设备共享同一条 I2C 总线，靠地址区分。

I2C 依赖你在第18章学过的**开漏输出**：任何设备都能主动拉低 SDA 线，但不会主动输出高电平（由外部上拉电阻提供）。这就是为什么 I2C 模块通常自带 4.7kΩ 上拉电阻——你一般不需要额外接线。

```python
from machine import Pin, SoftI2C

i2c = SoftI2C(scl=Pin(22), sda=Pin(21))
devices = i2c.scan()     # 扫描总线上的所有设备地址
print(devices)           # 如 [56, 60] → 0x38=AHT20, 0x3C=OLED
```

#### SPI——速度之王，I2C 的竞速对手

SPI（Serial Peripheral Interface）是 I2C 之外另一种常见协议。和 I2C 的"两根线分时共享"不同，SPI 给每个设备分配一根独立的选择线：

```
ESP32                         设备A         设备B
MOSI ────────────────────┬──    MOSI
MISO ────────────────────┼──    MISO
SCLK ────────────────────┼──    SCLK
CS0  ─────────────────────┤      CS
CS1  ─────────────────────┼──────────────    CS
```

- **MOSI**：主机输出，从机输入
- **MISO**：从机输出，主机输入
- **SCLK**：时钟线
- **CS**：片选线——每个设备一根，低电平表示"选中你"

**SPI vs I2C 关键区别：**

| 特性 | I2C | SPI |
|------|-----|-----|
| 线数 | 2（SDA+SCL） | 3+N（每个设备多一根 CS） |
| 设备数 | 最多 127（共享总线） | 受 CS 引脚数限制 |
| 速度 | 100k~400kbps（标准） | 最高几十 Mbps |
| 全双工 | ❌ 半双工 | ✅ 同时收发 |
| 是否需要地址 | ✅ 7位地址 | ❌ 靠 CS 线选择 |
| 典型用途 | 传感器、OLED、RTC | 屏幕、SD卡、Flash 存储 |

I2C 胜在简洁（两根线挂一切），SPI 胜在速度（适合高速大数据传输）。MicroPython 中 SPI 也用 `machine` 库：

```python
from machine import Pin, SPI
spi = SPI(1, baudrate=10_000_000, sck=Pin(18), mosi=Pin(23), miso=Pin(19))
```

对于本项目，OLED 和传感器都是 I2C 接口——两根线搞定所有，SPI 不需要用到。

#### UART——每个人的电脑都在用

UART（通用异步收发器）是串口通信的标准。ESP32 用它和电脑通信——你每次 `print()` 输出的内容，都是通过 UART 传到电脑的 USB 转串口芯片。

```
ESP32  TX  ─────  RX  电脑
       RX  ─────  TX
       GND ──── GND
```

UART 只需要**两根线（TX+RX）**，但只支持 1 对 1。它的特点：
- 最通用——几乎所有芯片都带 UART
- 不需要时钟线——双方约定相同的波特率（如 115200）就能通信
- 通过 RS-485/RS-232 可以传几十米

#### CAN 总线——汽车和工业的脊梁（简单了解）

CAN（Controller Area Network，博世 1986 年发明）是汽车和工业控制领域最主流的通信协议。你车里的发动机、刹车、空调、仪表盘之间全都通过 CAN 总线通信。

CAN 的核心思想：**多主模式**——总线上任何节点都能主动发消息，没有"主机"和"从机"的区分。消息带有优先级 ID，ID 越小优先级越高；两节点同时发消息时，高优先级的自动胜出，低优先级的自动退让——不需要软件仲裁。

ESP32 内置了 CAN 控制器（TWAI 模块），但本项目不涉及，只需知道它的存在。将来做机器人或车载项目时，CAN 会成为你的主力。

**四种协议速查：**

| 协议 | 线数 | 设备数量 | 速度 | 本章用途 |
|------|------|---------|------|---------|
| I2C | 2 | 最多127 | 100k~400k | OLED + 温湿度传感器 |
| SPI | 3+N | 受CS限制 | 最高几十M | （本章不用） |
| UART | 2 | 1对1 | 最高M | Shell调试输出 |
| CAN | 2 | 多主 | 最高1M | 汽车/工业（了解） |

### 21.2 AHT20 温湿度传感器（I2C 接口）

AHT20 是一款使用 I2C 接口的温湿度传感器，精度比 DHT11 高很多。它和 OLED 共享同一根 I2C 总线——只需要四根线（VCC、GND、SDA、SCL），两个设备都挂上去就行。

**硬件连接：**

```
AHT20       ESP32         OLED
VCC   ────  3.3V ────    VCC
GND   ────  GND  ────    GND
SDA   ────  GPIO21 ────  SDA
SCL   ────  GPIO22 ────  SCL
```

> AHT20 模块和 OLED 模块通常各自内置了 I2C 上拉电阻，直接并接即可，不需要额外元件。

在 Thonny → 工具 → 管理插件 中搜索安装 `ahtx0` 库。

### 21.3 OLED 显示屏（SSD1306）

0.96 寸 OLED 屏幕，分辨率 128×64 像素，使用 I2C 通信（I2C 协议详解见 21.1 节）。

**硬件连接：**

```
OLED        ESP32
VCC   ────  3.3V
GND   ────  GND
SCL   ────  GPIO22 (I2C 时钟)
SDA   ────  GPIO21 (I2C 数据)
```

在 Thonny → 工具 → 管理插件 中搜索安装 `ssd1306`。

### 21.4 完整代码

```python
from machine import Pin, SoftI2C
import ahtx0
import ssd1306
import time

# ====== I2C 初始化（AHT20 和 OLED 共享同一条总线）======
i2c = SoftI2C(scl=Pin(22), sda=Pin(21))

# ====== AHT20 初始化 ======
sensor = ahtx0.AHT20(i2c)

# ====== OLED 初始化 ======
oled = ssd1306.SSD1306_I2C(128, 64, i2c)

while True:
    try:
        # 读取温湿度（AHT20 的属性直接返回数值）
        temp = sensor.temperature       # 温度（℃）
        hum = sensor.relative_humidity  # 湿度（%）

        # 清屏
        oled.fill(0)

        # 显示标题
        oled.text("Temp & Hum", 0, 0)

        # 显示温度
        oled.text(f"Temp: {temp:.1f} C", 0, 25)

        # 显示湿度
        oled.text(f"Hum : {hum:.1f} %", 0, 45)

        # 下方提示
        oled.text("Press Ctrl+C", 0, 55)

        oled.show()
        time.sleep(2)

    except OSError:
        oled.fill(0)
        oled.text("Sensor Error!", 0, 25)
        oled.show()
        time.sleep(2)
```

### 21.5 运行效果

程序运行后，OLED 屏幕上会实时显示温湿度，每 2 秒更新一次。对着传感器哈一口气——湿度数值立刻上升。物理世界的数据就这样被你的代码"读"了出来！

注意看代码里 `sensor.temperature` 和 `sensor.relative_humidity` 后面**没有括号**——这是 AHT20 库用的一种叫"属性"的访问方式，和 `led.value()` 不一样。库作者选择了这种写法，你按库文档照用即可。
