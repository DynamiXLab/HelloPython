### 21.1 通讯协议——设备之间怎么"说话"

毕业项目要用到两个外接设备：DHT11 温湿度传感器和 OLED 屏幕。设备需要和 ESP32 **交换数据**——它们怎么通信？

嵌入式世界里，设备之间的通信有不同的"规则"（协议）。本章涉及两种最常见的：**单总线**和 **I2C**。顺便把 ESP32 另一个重要的 **UART** 也介绍一下。

#### I2C——两根线的优雅

OLED 屏幕用的就是 I2C 协议。I2C（Inter-Integrated Circuit，读作"I方C"或"I two C"）是一个叫 Philips 的公司在 1982 年发明的，只用**两根线**就能连接多个设备：

```
          SDA（数据线）— 所有设备共享
ESP32 ──┤
          SCL（时钟线）— 所有设备共享
         
          ┌─────────┐
          │  OLED   │ 地址 0x3C
          ├─────────┤
          │  DHT11  │（不是 I2C，单独讲）
          ├─────────┤
          │ 其他设备 │ 地址 0x68
          └─────────┘
```

- **SDA（Serial Data）**：数据线，双向传输数据
- **SCL（Serial Clock）**：时钟线，由 ESP32 发出节拍来同步通信节奏

每个 I2C 设备有一个**7 位地址**（像门牌号），ESP32 通过地址找到对应的设备通信。OLED 的地址通常是 `0x3C`。

I2C 能在两根线上挂很多设备，就是因为用了你在第18章学过的**开漏输出**——任何设备都能主动拉低 SDA 线，但不会主动输出高电平（由外部上拉电阻提供）。这是一种优雅的"线与"逻辑。

```python
from machine import Pin, SoftI2C

# I2C 初始化：SCL=GPIO22, SDA=GPIO21
i2c = SoftI2C(scl=Pin(22), sda=Pin(21))

# 扫描总线上所有设备的地址
devices = i2c.scan()
print(f"找到的 I2C 设备：{devices}")  # 如 [60] → 0x3C = 60
```

`SoftI2C` 中的 "Soft" 表示**软件模拟** I2C——用任意 GPIO 模拟 I2C 时序，不需要 ESP32 硬件 I2C 模块。优点是引脚自由选，缺点是速度比硬件 I2C 慢。对 OLED 应用来说完全够用。

#### 单总线（OneWire）——DHT11 的专利

DHT11 温湿度传感器用的不是 I2C，而是**单总线**（OneWire）协议——顾名思义，**只用一根数据线**完成所有通信（加上 VCC 和 GND 总共才三根线）。

```
ESP32          DHT11
3.3V   ────   VCC
GPIO4  ────   DATA（唯一的数据线）
GND    ────   GND
```

单总线用严格的时间间隔来区分 0 和 1——比如"拉低 50 微秒后拉高"代表 0，"拉低 50 微秒后保持拉高"代表 1。这些时序细节都由 DHT 库内部处理，你只需要调用 `dht_sensor.measure()`。

#### UART——最古老也最耐用的串口

UART（Universal Asynchronous Receiver/Transmitter，通用异步收发器）是**串口通信**的标准，比 I2C 还要老。ESP32 用它来和电脑通信——你每次 `print()` 输出在 Shell 里看到的东西，其实就是通过 UART 传到电脑的。

```
ESP32  TX  ────────────  RX  电脑（USB转串口芯片）
       RX  ────────────  TX
       GND ──────────── GND
```

UART 也是两根线（TX 发送、RX 接收），但**只支持一对一**通信，不像 I2C 可以挂多个设备。它的优点：
- 最通用：几乎所有的嵌入式设备都带 UART
- 最稳定：几十米的长距离传输也没问题（用 RS-232/RS-485）
- 不需要时钟线：双方约定好相同的波特率（如 115200）就能通信

**三种协议对比：**

| 特性 | I2C | OneWire | UART |
|------|-----|---------|------|
| 线数 | 2（SDA+SCL） | 1（DATA） | 2（TX+RX） |
| 设备数量 | 最多 127 个 | 通常 1 个 | 1 对 1 |
| 速度 | 100k~400kbps | 很慢 | 最高 Mbps 级别 |
| 是否需要地址 | ✅ 每个设备有地址 | ❌ 只有一根线 | ❌ 点对点 |
| 本章用途 | OLED 屏 | DHT11 传感器 | Shell 调试输出 |

### 21.2 DHT11 温湿度传感器

DHT11 是入门级温湿度传感器，精度一般但胜在便宜好用。

**硬件连接：**

```
DHT11       ESP32
VCC   ────  3.3V
DATA  ────  GPIO4
GND   ────  GND
```

在 Thonny → 工具 → 管理插件 中搜索安装 `dht` 库（通常内置，不需要额外安装）。

### 21.3 OLED 显示屏（SSD1306）

0.96 寸 OLED 屏幕，分辨率 128×64 像素，使用 I2C 通信（I2C 协议详解见 21.1 节）。

**硬件连接：**

```
OLED        ESP32
VCC   ────  3.3V
GND   ────  GND
SCL   ────  GPIO22 (I2C 时钟)
SDA   ────  GPIO21 (I2C 数据)
```

在 Thonny → 工具 → 管理插件 中搜索安装 `ssd1306`。

### 21.4 完整代码

```python
from machine import Pin, SoftI2C
import dht
import ssd1306
import time

# ====== DHT11 初始化 ======
dht_sensor = dht.DHT11(Pin(4))

# ====== OLED 初始化 ======
i2c = SoftI2C(scl=Pin(22), sda=Pin(21))
oled = ssd1306.SSD1306_I2C(128, 64, i2c)

while True:
    try:
        # 读取温湿度
        dht_sensor.measure()
        temp = dht_sensor.temperature()   # 温度（℃）
        hum = dht_sensor.humidity()       # 湿度（%）

        # 清屏
        oled.fill(0)

        # 显示标题
        oled.text("Temp & Hum", 0, 0)

        # 显示温度
        oled.text(f"Temp: {temp} C", 0, 25)

        # 显示湿度
        oled.text(f"Hum : {hum} %", 0, 45)

        # 下方面一条提示线
        oled.text("Press Ctrl+C", 0, 55)

        # 刷新屏幕
        oled.show()

        time.sleep(2)

    except OSError:
        oled.fill(0)
        oled.text("Sensor Error!", 0, 25)
        oled.show()
        time.sleep(2)
```

### 21.5 运行效果

程序运行后，OLED 屏幕上会实时显示当前的温度和湿度，每 2 秒更新一次。对着 DHT11 哈一口气，你会看到湿度数值上升——物理世界的数据就这样被你的代码"读"了出来！
