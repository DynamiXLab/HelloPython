### 附1 一个困扰：程序一关，数据就没了

回想一下你之前写的所有程序：你输入名字、程序输出问候，运行完就结束了。下次再运行，上次输入的名字还在吗？

```python
# 试试看：运行一次，输入数据，关掉，再重新运行
name = input("请输入你的名字：")
```

第二次运行时，`name` 又回到空的状态——程序重新开始了。数据只在程序运行时存活，一关就丢。

这称为**易失性**——内存里的数据靠电维持，断电就没了。你需要一个能把数据**存到磁盘上**的方法，这样下次打开程序还能读到。这就是**文件操作**要做的事。

### 附2 写入文件——把数据存到磁盘上

写文件的最基本方式是 `open()` + `write()`：

```python
# 打开一个文件准备写入
f = open("mydata.txt", "w", encoding="utf-8")
f.write("第一行内容\n")
f.write("第二行内容\n")
f.close()    # 用完一定要关闭！
```

运行完这段代码，你会在当前文件夹里看到一个叫 `mydata.txt` 的文件。用记事本打开它，里面有两行字——关掉 Python 它们还在。

**但是用这种方式容易忘掉 `f.close()`**，忘记关闭文件可能导致数据没写完，或者占用系统资源。所以 Python 提供了一个更安全的写法——`with` 语句：

```python
# with 会在代码块结束后自动关闭文件
with open("mydata.txt", "w", encoding="utf-8") as f:
    f.write("第一行内容\n")
    f.write("第二行内容\n")
# 到这里 f 自动关闭了，不需要手动 close()
```

> 推荐永远用 `with` 写法——它比你更不容易忘记关文件。

**`"w"` 模式是什么意思？**

`open()` 的第一个参数是文件名，第二个参数是**打开模式**：

- `"w"`：write，写入模式。如果文件已存在，**会覆盖原有内容**从头写；如果文件不存在，会自动创建

```python
with open("test.txt", "w", encoding="utf-8") as f:
    f.write("第一轮写入")

with open("test.txt", "w", encoding="utf-8") as f:
    f.write("第二轮写入")    # test.txt 里只有"第二轮写入"，第一轮的被覆盖了
```

第三个参数 `encoding="utf-8"` 是告诉 Python 用 UTF-8 编码来保存文本——这样中文才能正确存储。写文件时永远加上它。

### 附3 读取文件

```python
# 一次性读取全部内容
with open("mydata.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print(content)

# 按行读取
with open("mydata.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())   # strip() 去掉末尾的换行符
```

### 附4 追加内容

```python
# 'a' 模式：追加（不会覆盖，写在文件末尾）
with open("mydata.txt", "a", encoding="utf-8") as f:
    f.write("这是追加的新一行\n")
```

| 模式 | 含义 | 文件不存在时 |
|------|------|-------------|
| `'r'` | 只读 | 报错 |
| `'w'` | 写入（覆盖） | 自动创建 |
| `'a'` | 追加 | 自动创建 |

💡 **思考延续**

你有没有想过一个基本问题：为什么磁盘里存的东西要分成一个个"文件"？直接全堆在一起不行吗？

事实上早期的计算机就是这么干的。在 1950~60 年代，数据要么存在打孔卡上（一摞卡片就是一个"文件"），要么存在磁带上（一盘磁带就是一个"文件"）。磁带的问题是：你要读到末尾的数据，必须从头卷到末尾，中间不能跳。这就像想读一本书的第 300 页，却必须从第 1 页开始翻起。

"文件系统"的概念在 1960 年代的操作系统中诞生——它把磁盘可能分成小块（称为"数据块"），然后用一个"目录"（类似书的目录页）来记录每个文件名对应哪些数据块。从此，程序只要告诉操作系统"我要读文件 X"，操作系统就能根据目录跳到对应的位置，不用从头扫到尾。数据第一次有了"地址"——也就是你熟悉的 `C:\Users\小明\Desktop\notes.txt` 这种路径。

Python 的 `with open("mydata.txt", "w") as f:` 在你看来只是一行语法——但它背后是几十年的操作系统演进。`with` 会保证文件用完自动关闭，这是在帮你避免 50 年前程序员最常犯的一个错误：打开了一百个文件，关闭了九十九个，漏掉的那一个占用了操作系统的资源配额，导致程序后面崩溃。一个 `with` 关键字，替你省去了一个可以让人通宵排查的 bug。
