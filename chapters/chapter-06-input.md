### 6.1 获取用户输入

```python
name = input("请输入你的名字：")
print(f"你好，{name}！")
```

运行时，程序会停下来等你输入，输入完成后按回车继续。

### 6.2 重点：input() 返回的是字符串！

```python
age = input("请输入年龄：")
print(type(age))   # <class 'str'> —— 是字符串！
print(age + 1)     # ❌ 报错！字符串不能加数字
```

所以需要类型转换：

```python
age = int(input("请输入年龄："))     # 一步到位转成整数
print(f"明年你将是{age + 1}岁")
```

### 6.3 结合前面所学的

现在我们可以写一个完整的交互小程序了：

```python
# 小小计算器
num1 = int(input("请输入第一个数字："))
num2 = int(input("请输入第二个数字："))
print(f"{num1} + {num2} = {num1 + num2}")
print(f"{num1} × {num2} = {num1 * num2}")
```

💡 **思考延续**

你现在可以舒服地坐在电脑前，用键盘输入数据，程序即时响应——但在 1950 年代，"输入"完全是另一回事。程序员写代码用的不是键盘，而是**打孔卡**：一张近似明信片大小的纸卡，按固定位置打上孔或留空，代表一行代码或一个数据。一个程序动辄几百张、上千张卡片。

如果哪天你不小心把一摞卡片撒在地上——对不起，你只能靠卡片侧面预先画好的箭头标记一张一张重新排序，没有任何"撤销"或"自动恢复"。更糟的是，你排好序提交给计算机之后，可能要等好几个小时甚至第二天才能拿到结果。如果中间有一张卡片放错了位置——从头再来。

在这种背景下，`input()` 的诞生是一次革命：它让程序可以在运行中途停下来，**等待人类实时回应**，而不是把所有输入提前打在卡片上一次性交给机器。1950 年代的程序员如果看到你现在写的 `name = input("请输入名字：")`，大概会感动到落泪。

你现在觉得理所当然的"交互式编程"，是前人付出了几代人的努力才变成现实的。
