### 3.1 什么是变量

变量就像一个**贴了标签的盒子**：你把数据放进去，以后用标签（变量名）就能找到它。

```python
name = "小明"      # 把 "小明" 放进名叫 name 的盒子里
age = 12            # 把 12 放进名叫 age 的盒子里
print(name)         # 输出：小明
print(age)          # 输出：12
```

### 3.2 变量命名规则

- 只能用字母、数字、下划线 `_`
- 不能以数字开头
- 区分大小写：`Name` 和 `name` 是两个不同变量

```python
my_name = "张三"    # ✅ 正确
_name = "李四"      # ✅ 正确
score2 = 95         # ✅ 正确
2score = 95         # ❌ 不能以数字开头
```

### 3.3 变量的值可以变化

```python
count = 10
print(count)   # 输出：10
count = 20     # 重新赋值
print(count)   # 输出：20
count = count + 1  # 在原来的基础上加 1
print(count)   # 输出：21
```

### 3.4 小心！不同类型不能混用

回头看 3.1 的两个变量：

```python
name = "小明"    # 字符串（文本）
age = 12         # 整数（数字）
```

它们虽然是变量，但**存储的数据类型不一样**。Python 里不同类型之间不能随便混在一起操作——试一下：

```python
print(name + age)    # ❌ 会报错！
```

你会看到这样的错误信息：

```
TypeError: can only concatenate str (not "int") to str
```

意思是：字符串只能拼接字符串，不能拼接整数。

那如果想一起使用怎么办？两种方式：

**方式一：用 `str()` 转换**

`str()` 是一个**类型转换函数**，作用是把括号里的数据**转成字符串**（文本形式）。

```python
print(name + str(age))       # 小明12
print(name + "今年" + str(age) + "岁")  # 小明今年12岁
```

`str(12)` 把数字 `12` 变成了字符串 `"12"`，这样就可以和前面的字符串拼接了。

**方式二：用第5章会学的 f-string（推荐）**

```python
print(f"{name}今年{age}岁")  # 小明今年12岁
```

f-string 会自动把变量转换成字符串，不需要手动调用 `str()`。

> **小贴士**：看到 `TypeError` 不要慌，它只是告诉你"类型搞混了"。在编程里看到报错很正常，学会看错误信息就能快速找到问题！

### 3.5 常用的类型转换函数

Python 提供了几个内置函数来转换数据类型：

| 函数 | 作用 | 示例 | 结果 |
|------|------|------|------|
| `str()` | 转成字符串（文本） | `str(100)` | `"100"` |
| `int()` | 转成整数 | `int("18")` | `18` |
| `float()` | 转成小数 | `float("3.14")` | `3.14` |

这些函数**不会修改原变量**，而是返回转换后的新值。例如：

```python
age = "18"                # 字符串
print(age + 1)            # ❌ TypeError：字符串不能加数字

age_num = int(age)        # 转换成整数，存到新变量里
print(age_num + 1)        # ✅ 19

price = "9.99"
price_float = float(price)
print(price_float * 2)    # ✅ 19.98
```

💡 **思考延续**

你有没有想过一个问题——`print(name)` 输出的是 `小明`，不带引号，干干净净。但你的代码里写的是 `name = "小明"`，那对引号去哪儿了？

答案是：引号从来就不是字符串的一部分。引号是 Python 用来**区分"这是字符串"和"这是变量名"** 的语法标记。当代码运行时，Python 读取到 `"小明"`，知道这是一个字符串，于是剥掉引号，把里面的内容 "小明" 存入内存；当你 `print(name)` 时，它从内存取出来原样显示——所以没有引号。

同样一幅图更有趣：数字 `12` 和字符串 `"12"` 在屏幕上看起来一模一样，但内存里的存储方式天差地别。数字 `12` 用二进制存——`1100`——共 4 个比特。字符串 `"12"` 则用字符编码存——`0x31 0x32`——每个字符占一个字节。所以 Python 坚决不让 `"小明" + 12` 通过：一个是文本编码流，一个是二进制数，硬加在一起毫无意义。这个"不同种类不能混用"的原则，不是 Python 在刁难你，而是计算机构造层面的铁律。

把视野再拉远一点：在 Python 的世界里，数字、字符串、列表、字典……**一切都是对象**。每个对象内部都规定了"我的数据该怎么存"和"你能对我做什么操作"。你现在学到的变量名，本质上只是一个贴在对象上的标签——对象本身在内存里过日子，标签只是让你能找到它。同一个对象可以贴多个标签（多个变量指向同一个值），后面学列表的时候你就会亲身体验到这一点。
