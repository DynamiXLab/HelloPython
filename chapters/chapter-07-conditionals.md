### 7.1 if 语句

```python
score = 85
if score >= 60:
    print("及格了！")
```

#### Python 的缩进规则（必读，这是重点）

注意到了吗？`if` 那行以冒号 `:` 结尾，而下一行 `print("及格了！")` 前面**空了一段距离**——这个空白叫**缩进**。在 Python 里，缩进不是装饰，而是语法的一部分。

**缩进的作用：告诉 Python 哪些代码属于这个 if**

```python
score = 85
if score >= 60:
    print("及格了！")          # ← 缩进：这行属于 if
    print("继续加油！")        # ← 同样缩进：也属于 if
print("程序结束")              # ← 没有缩进：不属于 if
# 输出：
# 及格了！
# 继续加油！
# 程序结束
```

如果 `print("程序结束")` 也缩进了，它就会变成 if 的一部分——只会在及格的条件下执行。**缩进决定了代码的"归属"**。

**缩多少？**

- 官方推荐：**4 个空格**（每次缩进按一下 Tab 键——好的编辑器会自动转换成 4 个空格）
- 可以少缩吗？可以缩 2 个、3 个、5 个……但一个文件里**必须保持一致**
- 可以多缩吗？同一层级必须缩同样的数量

```python
score = 85
if score >= 60:
    print("及格了！")       # ✅ 缩进 4 个空格
      print("继续加油！")   # ❌ 缩进 6 个空格——不一致，会报错！
```

**为什么 Python 要这样设计？**

其他很多语言（C、Java、JavaScript）用花括号 `{}` 来表示代码块：

```c
if (score >= 60) {
    printf("及格了！");    // 花括号里的内容属于 if
    printf("继续加油！");
}
```

Python 的设计者 Guido van Rossum 认为：花括号是多余的——既然好的程序员本来就会用缩进让代码可读，不如直接让缩进成为语法。这样强制每个人写出格式统一的代码。Python 代码看起来干净、整齐，很大程度上就是缩进规则的功劳。

**常见的缩进错误**

```python
# 错误 1：忘了缩进
if score >= 60:
print("及格了！")           # ❌ IndentationError: expected an indented block

# 错误 2：混用 Tab 和空格
if score >= 60:
→   print("及格了！")       # ← Tab
    print("继续加油！")     # ← 4 个空格  ❌ 混用会报错！

# 错误 3：同一层级缩进不一致
if score >= 60:
    print("及格了！")
      print("继续加油！")   # ❌ 多缩了 2 格，Python 以为这是另一层
```

> **Thonny 会自动帮你**：在 Thonny 里，你在 `if` 那行末尾按回车，下一行会自动缩进。你只需要正常打字就好——Thonny 会帮你管理缩进。如果复制了网上的代码导致缩进混乱，选中全部代码按 `Tab` 或 `Shift+Tab` 统一调整。

### 7.2 if...elif...else

```python
score = 85

if score >= 90:
    print("优秀")
elif score >= 80:
    print("良好")
elif score >= 60:
    print("及格")
else:
    print("不及格")

# 输出：良好
```

程序从上到下检查，碰到第一个满足的条件就执行对应的代码块，后面的都不再检查。

### 7.3 比较运算符

| 运算符 | 含义 |
|--------|------|
| `==` | 等于（注意是两个等号！） |
| `!=` | 不等于 |
| `>` | 大于 |
| `<` | 小于 |
| `>=` | 大于等于 |
| `<=` | 小于等于 |

### 7.4 多个条件组合：and / or

```python
age = 20
has_ticket = True

if age >= 18 and has_ticket:
    print("可以入场")
else:
    print("不能入场")

# and：两边都满足才是 True
# or：任意一边满足就是 True
```

💡 **思考延续**

`True` 和 `False` 这两个词背后藏着一位 19 世纪英国数学家的故事。

1854 年，一位自学成才的数学家 **George Boole** 出版了一本名为《思维规律的研究》的书。他在书中提出了一个在当时看来有些疯狂的想法：**人类的逻辑推理可以用数学符号来表达和计算**。比如"如果今天下雨，那么路会湿"——Boole 说可以把"下雨"标记为 A、"路湿"标记为 B，然后整个推理就变成了"如果 A 为真，则 B 为真"。

这个想法在当时几乎没什么实际用途。Boole 本人 49 岁就去世了（因为在大雨中走了三英里去上课，感染肺炎），他大概不会想到，一个半世纪后，他发明的这套"布尔代数"会成为人类有史以来最庞大机器的核心。

今天的每一台计算机里，数以亿计的晶体管每秒执行着数十亿次布尔运算——`and`、`or`、`not`、比较大小、判断相等。你在代码里写的每一行 `if...elif...else`，最终都被翻译成了一连串的布尔运算，在硅片上无声地飞驰。Boole 的遗产不在博物馆里——它在你敲下的每一行条件的背后。
