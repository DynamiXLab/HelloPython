### 13.1 一个难题：重复写同样的代码

假设你想给三个朋友打招呼。你会怎么写？

```python
print("你好，张三！欢迎学习Python！")
print("你好，李四！欢迎学习Python！")
print("你好，王五！欢迎学习Python！")
```

如果改成用英文打招呼呢？你得把三行都改一遍。如果有 10 个朋友要打招呼呢？复制粘贴十次，字体稍微歪一点就乱了。

再想象你要计算圆的面积 5 次——每次都得写 `3.14 * r ** 2`，改一次半径就要复制一行出来改。你有没有一种感觉：**代码在重复自己，一定有什么办法可以只写一次逻辑，重复使用**？

### 13.2 定义和调用函数——把代码"打包"

**函数**就是来解决这个问题的。它让你**把一段代码打包，给它取个名字，之后用名字就能运行这段代码**。

定义函数的关键字是 `def`（define 的缩写），后面跟函数名和冒号：

```python
def greet():
    print("你好！")
    print("欢迎学习 Python！")
```

写完这段代码，屏幕上什么都没出现。因为定义函数只是在"打包"，并没有运行。要在需要的时候运行它，你得**调用函数**：

```python
greet()   # 运行 greet() 里的代码
greet()   # 再运行一次
```

输出：
```
你好！
欢迎学习 Python！
你好！
欢迎学习 Python！
```

只定义了一次 `greet`，调用了两次，里面的两行 `print` 就执行了两次。

> 为什么定义函数时什么都不显示，调用时才有输出？可以把函数想象成一个烤箱——定义烤箱是造一个能烤东西的机器（只是造好，没有烤），调用函数才是开机烤面包。

### 13.3 参数——让函数"收件"

上面的 `greet()` 对谁都说同样的话。如果想让它对不同的人说不同的话呢？你需要给函数**传递信息**——用**参数**（parameters）。

```python
def greet(name):
    print(f"你好，{name}！欢迎学习Python！")

greet("张三")    # 你好，张三！欢迎学习Python！
greet("李四")    # 你好，李四！欢迎学习Python！
greet("王五")    # 你好，王五！欢迎学习Python！
```

对比之前的写法——同样的功能，现在只写了一行 `print`，定义了 4 行代码，就能对任意多人打招呼。

**参数是怎么工作的？**

`def greet(name):` 这一行里，`name` 是一个**变量占位符**。当你调用 `greet("张三")` 时，Python 做了两件事：
1. 把 `"张三"` 放进 `name` 这个变量里
2. 然后运行函数体内的代码，这时 `name` 已经等于 `"张三"` 了

```python
# 调用 greet("张三") 时，Python 内部像在执行：
name = "张三"
print(f"你好，{name}！欢迎学习Python！")
```

**多个参数**

函数可以接收多个参数，用逗号隔开：

```python
def greet(name, times):
    print(f"你好，{name}！" * times)

greet("张三", 3)   # 你好，张三！你好，张三！你好，张三！
```

调用时，参数的顺序必须和定义时一一对应——第一个值给第一个参数，第二个值给第二个参数。

### 13.4 返回值——让函数"算出结果"

前面的函数都是在内部直接 `print` 输出。但很多时候，你需要函数算出一个结果，交给调用它的代码继续处理——而不是直接打印出来。

```python
def circle_area(radius):
    result = 3.14 * radius ** 2
    return result    # 把结果"还回去"

a = circle_area(5)   # 把返回值存到变量 a 里
print(a)             # 78.5
b = circle_area(10)
print(b)             # 314.0
```

**`return` 做了两件事：**
1. 把后面的值**返回**给调用它的地方——`circle_area(5)` 本身变成了 `78.5`
2. 函数**立即结束**——`return` 后面的代码不会再运行

```python
def test():
    return "结束"
    print("这行永远不会运行")   # ❌ return 之后不会再执行
```

**没有 `return` 会怎样？**

```python
def no_return():
    x = 1 + 2

result = no_return()
print(result)   # None（表示"什么都没有"）
```

没有 `return` 的函数，调用时会返回 `None`（Python 里表示"空"或"没有"的词）。

**一个重要的区别：`print` vs `return`**

```python
def with_print(x):
    print(x * 2)

def with_return(x):
    return x * 2

a = with_print(5)     # 屏幕显示 10，但 a 是 None
b = with_return(5)    # 屏幕什么都不显示，但 b 是 10
print(b + 3)          # 13
```

`print` 是把结果显示在屏幕上（仅此而已）；`return` 是把结果**还给调用者**，让变量可以接住它，继续参与后面的运算。

### 13.5 多参数 + 多返回值

函数可以接收多个参数，也可以返回多个值：

```python
def rectangle(length, width):
    area = length * width
    perimeter = 2 * (length + width)
    return area, perimeter

mianji, zhouchang = rectangle(10, 5)
print(f"面积：{mianji}，周长：{zhouchang}")
```

`return area, perimeter` 返回的是**两个值打包成一个元组**。接收时用 `mianji, zhouchang =` 分别接住（这叫"解包赋值"）。

💡 **思考延续**

这一章你学会了用函数来打包代码，方便重复使用。在软件工程界，有一条与此相关的著名原则叫 **DRY**——Don't Repeat Yourself（不要重复自己）。

这条原则来自 Andy Hunt 和 Dave Thomas 在 1999 年出版的《程序员修炼之道》——一本被无数程序员称为"必读"的书。书中有个比喻：如果你在程序里把同一段逻辑写了两次，那就等于把一个问题的答案同时写在两个地方。如果某天问题变了（需求改了、bug 要修），你只改了第一个地方而漏了第二个——恭喜你，你刚制造了一个新 bug。

这个"修改不同步"的 bug 是最隐蔽、最难追查的类型之一。因为代码在两个不同地方各执行一次，单独看每一处都没问题，组合起来却互相矛盾。工程师花在追踪这种不一致上的时间，多到足以让 DRY 原则成为行业圣经。

所以函数不只是"让代码短一点"的工具——它是一个**单点控制**（single point of control）：你的圆的面积计算逻辑写一次，所有调用它的地方自动生效；哪天要改 `3.14` 为 `3.14159`，改一个地方就够了。"少写一行代码"不是偷懒，是深谋远虑。
